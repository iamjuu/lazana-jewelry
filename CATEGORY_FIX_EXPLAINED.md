# Category Slug Generation - Issue Fixed! ✅

## 🐛 The Problem

**Error:** `Category validation failed: slug: Path 'slug' is required.`

## 🔍 Root Cause

MongoDB schema validation happens **BEFORE** the `pre('save')` hook runs.

```
Flow:
1. new Category({ name: "Crystal Bowls" })  ← No slug provided
2. ❌ Schema validation: "slug is required"  ← FAILS HERE
3. ❌ pre('save') hook never runs            ← Never reached
```

## ✅ The Solution

Changed slug field from `required: true` to `required: false`:

```typescript
slug: {
  type: String,
  required: false,  // ✅ Generated by pre-save hook
  unique: true,
  lowercase: true,
  sparse: true,     // ✅ Allow null for unique index
}
```

Now the flow works:
```
1. new Category({ name: "Crystal Bowls" })
2. ✅ Schema validation passes (slug optional)
3. ✅ pre('save') hook runs
4. ✅ Hook generates slug: "crystal-bowls"
5. ✅ Category saved successfully
```

## 🎯 How It Works Now

### **Pre-Save Hook Logic:**

```typescript
CategorySchema.pre('save', function (next) {
  // Step 1: For new docs or modified names
  if (this.name && (this.isNew || this.isModified('name'))) {
    this.name = toTitleCase(this.name.trim());
    this.slug = generateSlug(this.name);
  }
  
  // Step 2: Safety fallback
  if (!this.slug && this.name) {
    this.slug = generateSlug(this.name);
  }
  
  next();
});
```

### **API Route:**

```typescript
// Create category
const category = new Category({
  name: "crystal bowls",
  description: "..."
});

await category.save();  // ✅ Triggers hook
// Result: { name: "Crystal Bowls", slug: "crystal-bowls" }
```
